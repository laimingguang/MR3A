# 配置说明

建议先跟随软件内置教程快速了解各个功能。

教程位置：MR3A软件左下角「设置」→「关于我们」→ 页面底部「使用教程」。

下面简单介绍一些重要的按钮功能和配置。

## 主界面

### 重新连接 / 刷新

软件右上角的两个小按钮，点击可重新连接模拟器或刷新界面。实在连不上可以重启 MR3A，能解决 96% 的问题（确信）。

### 任务列表

右键任务可单独运行、复制、粘贴等；点击列表上方加号可添加新任务。

### 添加实例

顶部标签页点击加号，可添加任务组，可自行调整任务，适用于大号/小号分离、活动/日常分开等场景。

## 设置

点击软件左下角「设置」进入设置页面。

### 定时执行

定时执行是指定时执行**脚本任务**，而非定时启动 MR3A 软件本身，因此需要 MR3A 常驻后台。

### 启动设置

1. **启动后操作**：指启动 MR3A 后自动执行的动作，可选 `仅启动软件` 或 `启动软件并运行脚本`。这里的"软件"指模拟器，"脚本"指 MR3A 的任务。
2. **软件路径**：填对应模拟器的可执行文件路径。

    | 模拟器 | 路径 |
    | :--- | :--- |
    | MuMu12（旧） | `{安装目录}\shell\MuMuPlayer.exe` |
    | MuMu12 V5（新） | `{安装目录}\nx_device\12.0\shell\MuMuNxDevice.exe` |
    | 雷电 9 | `{安装目录}\dnplayer.exe` |

3. **附加命令**：多开模拟器时填写，用于指定启动哪个实例。

    | 模拟器 | 命令格式 | 示例 |
    | :--- | :--- | :--- |
    | MuMu | `-v 多开号` | `-v 0` |
    | 雷电 | `index=多开号` | `index=0` |

    > 多开号从 0 开始，0 代表第一个多开模拟器，依次类推。

### 连接设置

`ADB 连接失败时尝试启动模拟器`：开启后，点击运行任务时若 ADB 连接失败，会从上述[软件路径](#启动设置)中启动模拟器并重试连接。

### 更新设置

`下载源`推荐选择 **GitHub**，暂不支持 Mirror 酱。
