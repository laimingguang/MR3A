# 使用场景

## 1. 启动后自动执行任务

在「设置 → 启动设置 → 启动后操作」中配置。这里的 `启动软件` 指启动模拟器，`启动脚本` 指运行 MR3A 的任务；选择 `启动软件并运行脚本`，打开 MR3A 后便会自动启动模拟器并开始执行任务。

::: warning 注意
需正确填写模拟器路径，详见[软件路径](config.md#启动设置)。
:::

## 2. 任务结束后自动操作

在「设置 → 启动设置 → 结束后操作」中选择对应操作：关机、关闭模拟器或关闭程序等。

也可以在任务列表中添加"关机"任务，实现不同配置下的灵活收尾。

## 3. 定时执行

定时执行需要 MR3A 常驻后台（可隐藏到系统托盘）。

### 场景 A：模拟器已启动（推荐）

在「设置 → 定时执行」中配置定时时间，并开启「自定义配置选择」，即可按不同时间执行不同配置的任务。

### 场景 B：模拟器未启动

需先确保 MR3A 能自动启动模拟器：

1. 在[启动设置](config.md#启动设置)中正确填写`软件路径`和`附加命令`。
2. 开启「设置 → 连接设置 → ADB 连接失败时尝试启动模拟器」，该选项会读取`软件路径`并启动模拟器。
3. 参照场景 A 配置定时执行时间。

## 4. 单模拟器多账号依次运行

在上方标签栏点击「添加实例」，按 **启动游戏 → 任务 → 关闭游戏 → 启动游戏 → 任务 → 关闭游戏** 的顺序排好任务并配置相应参数即可。

## 5. 多模拟器同时运行

若需多开模拟器同时操作，可将 MR3A 文件夹复制多份，使用不同的 MR3A、同一个 adb.exe、不同的连接地址来连接。
